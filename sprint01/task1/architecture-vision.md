# Архитектурное видение проекта

**Дата**: 07 мая 2026 г.

## Задача

Переход компании «Diamond Universe» к модели онлайн аукционов.

### Проблема

В условиях пандемии традиционные методы продаж стали неэффективны.
Резкое сокращение свадеб, поездок и перемещений из-за самоизоляции людей привело
к сокрушительному падению объёмов реализации.

Требуется создать процесс удалённой покупки оцифрованных алмазов, сохранив высокое
доверие к качеству продукции.

### Кому помогает решение

- **Компании**. Откроется новый рынок сбыта, позволяющий масштабировать продажи.
- **Клиентам** — сотням компаний из Индии, Китая, Бельгии и других мировых центров
торговли алмазами. Появится удобный способ оценить и купить ценные активы в любое время
из любой точки мира без личного визита.

### Важность задачи

Предлагаемое решение критически важно для
- восстановления прибыли;
- минимизации рисков, связанных с физическими ограничениями;
- обеспечения устойчивости бизнеса в условиях меняющегося рынка.

## Контекст

### Почему задача возникла сейчас

Пандемия 2019-2020 годов продемонстрировала уязвимость офлайн-модели.
Для выживания компании необходимо как можно скорее внедрить цифровой канал продаж.

### Текущее состояние

Компания опирается на офлайн-аукционы, проводимые в офисе.
Процессы автоматизированы в рамках существующих систем, не предназначенных
для работы с удалёнными покупателями.
Процесс передачи данных между офисом, лабораторией и клиентами требует новых механизмов.

### Целевое состояние

Полностью автоматизированна экосистема цифровых аукционов.
В этой системе данные о камнях бесшовно передаются от лаборатории к клиенту.
Процесс отбора, логистики и оплаты интегрирован в единый контур.
Риск человеческой ошибки, связанный с ручным вводом, сведён к минимуму.

### Ограничения

- **Временной фактор**. Первый цифровой аукцион должен быть проведён через **два месяца**.
Таким образом, исключается возможность полной переработки систем к моменту
проведения первого аукциона.

- **Технологический разрыв**. Необходимо работать с внешними системами (например, лаборатория)
и внешними облачными сервисами на переходном этапе.

## Описание оптимального решения

Из-за сжатых сроков для решения задачи предлагается стратегия двухэтапного перехода:
1. Быстрый старт (промежуточное решение).
2. Масштабирование (целевое решение).

### Основные этапы проекта

#### Этап 1: запуск цифрового канала (0-2 месяца)

**Метод**: использование ручной цифровизации.

**Действия**:
подготовка данных в Excel по утверждённым шаблонам;
использование текущих систем для логистики и планирования;
заключение контракта с провайдером частного облака для публикации данных клиентам.

**Результат**: проведение первого успешного онлайн-аукциона.

#### Этап 2: автоматизация и интеграция (2-12 месяцев)

**Метод**: глубока доработка ИС; создание интеграционных шлюзов.

**Действия**:
разработка модулей в ИС Алмазы и ИС Аукцион для отказа от Excel;
настройка автоматической передачи данных из ИС Лаборатория;
автоматизация взаимодействия с ИС Бухгалтерия и ИС Логистика.

**Результат**: сквозной автоматизированный процесс.

### Что понадобится для реализации

- Персонал:
    * системные аналитики для проектирования схем передачи данных;
    * команда разработчиков (backend, frontend);
    * QA-инженеры для тестирования процессов;
    * специалисты по логистике.
- Технологический стек:
    * инструменты для работы с облачными сервисами;
    * средства интеграции для связи внутренних и внешних систем.
- Бизнес-ресурсы:
    * контракты с лабораторией и транспортными компаниями, адаптированные
    под цифровые процессы.
