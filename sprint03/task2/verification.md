# Описание точки верификации

Сервис сначала выполняет детерминированное сопоставление по валидному уникальному
штрихкоду. Для оставшихся SKU AI-сервис возвращает

- кандидата;
- оценку уверенности;
- признаки, которые повлияли на результат.

Автоматически принимается совпадение только при одновременном выполнении условий:

- оценка уверенности не ниже 95%;
- отсутствуют конфликтующие штрихкоды;
- совпадают категория, единица измерения и размер упаковки;
- цена не выходит за допустимый диапазон категории;
- кандидат единственный.

Все остальные позиции направляются товароведу вместе с тремя лучшими кандидатами.

Дополнительно товаровед проверяет случайную выборку 2% автоматически принятых
позиций. Если доля ошибок в выборке выше 1%, пакет блокируется и возвращается на
расширенную проверку всего пакета. Только после завершения проверки SKU Matching
Service передаёт подтверждённое сопоставление в SAP ERP. Публикация каталога в Cloud
POS выполняется после подтверждения успешной загрузки сопоставления товаров в SAP.

## Контроль SLA и качества

| Метрика | Целевое значение |
| --- | --- |
| Полный онбординг магазина | не более 5 рабочих дней |
| Доля автоматически обработанных SKU после стабилизации модели | не менее 70% |
| Ошибочные автоматические сопоставления | менее 1% |
| Необработанные позиции перед публикацией | 0 |
| Аудит решений | 100% сопоставлений |

Решения товароведа сохраняются как размеченные данные для последующего улучшения
модели, но новая версия модели вводится только после офлайн-оценки и одобрения.
