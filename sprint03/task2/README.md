# Задание 2. Целевой процесс онбординга магазина

## Цель процесса

Подключить новый магазин не более чем за пять рабочих дней, исключив email,
ручной VLOOKUP и просмотр всего ассортимента человеком. SAP ERP остаётся
эталонным источником номенклатуры, а Cloud POS получает только подтверждённый
каталог.

## Артефакты

- [store-onboarding.bpmn](store-onboarding.bpmn) — нормативная BPMN 2.0 модель
  процесса.
- [verification.md](verification.md) — описание точки верификации.

## Участники и ответственность

| Участник | Ответственность |
| --- | --- |
| M&A / Store Ops | Загружает и при необходимости исправляет пакет SKU, получает итоговый отчёт |
| SKU Matching Service | Валидирует и нормализует данные, оркестрирует совпадения, публикацию и аудит |
| AI Service | Возвращает кандидатов SKU, признаки и оценку уверенности |
| Товаровед / Data Integration | Проверяет исключения, выборку и при необходимости весь пакет |
| SAP ERP | Принимает подтверждённый маппинг |
| Cloud POS | Публикует подтверждённый каталог нового магазина |

## Ключевые ветки

- Невалидный пакет возвращается Store Ops для исправления и повторной загрузки.
- Совпадения, прошедшие оценку уверенности и другие правила, принимаются автоматически.
- Исключения и выборка 2% передаются товароведу.
- При ошибке выборки выше 1% товаровед проверяет весь пакет.
- Каталог передаётся в Cloud POS только после подтверждения загрузки маппинга в
  SAP ERP.
- Store Ops завершает онбординг только после подтверждения публикации и получения
  итогового отчёта.
