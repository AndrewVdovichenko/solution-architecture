# Расчёт TCO и ROI

## Методика и допущения

Расчёт выполнен в ценах 2026 года без дисконтирования и инфляции для 100
магазинов. В кейсе нет тарифов и данных ФОТ, поэтому значения ниже являются
проверяемыми допущениями для принятия архитектурного решения.

| Параметр | Допущение |
| --- | --- |
| Горизонт | 3 года |
| Количество магазинов | 100 |
| Полная стоимость одного ИТ-специалиста | 2,4 млн руб./год |
| As-Is поддержка | 12 FTE |
| To-Be поддержка | 4 FTE |
| Лицензии Windows/SQL и локального ПО | 150 тыс. руб./магазин/год |
| Cloud POS SaaS | 72 тыс. руб./магазин/год |
| Облачная платформа | 4,8 млн руб./год на 100 магазинов |
| Связь As-Is | 36 тыс. руб./магазин/год |
| Основной и резервный интернет To-Be | 48 тыс. руб./магазин/год |
| Миграция, интеграция и обучение | 15 млн руб. однократно |

As-Is обновление железа распределено по трём годам, так как парк разновозрастный:
10 млн руб. в первый год, 8 млн во второй и 7 млн в третий. To-Be требует в
первый год 10 млн руб. на планшеты, Edge Gateway и подготовку ККТ.

## Экономический эффект

Трёхлетняя экономия:

`200,8 − 107,8 = 93,0 млн руб.`

Снижение TCO:

`93,0 / 200,8 × 100% = 46,3%`

Выгода до учёта разовой миграции:

`200,8 − (107,8 − 15,0) = 108,0 млн руб.`

ROI миграции:

`(108,0 − 15,0) / 15,0 × 100% = 620%`

Консервативный срок окупаемости рассчитан без учёта экономии на обновлении серверов.
Регулярные расходы снижаются с 58,6 до 27,6 млн руб. в год:

`15,0 / ((58,6 − 27,6) / 12) = 5,8 месяца`

## Вывод

Миграция выгодна.
Для реализации расчётного эффекта необходимо завершать каждую волну миграции отключением локального
сервера и лицензий, а также ежемесячно сверять фактические SaaS/PaaS расходы с
моделью.
