# Задание 1. Архитектурная диагностика и целевое видение

В разделе описан переход AdScale от монолита к целевой микросервисной архитектуре,
способной выдержать интеграцию с новой DSP-площадкой.

## Состав артефактов

- [AS-IS.md](AS-IS.md) — диагностика текущей архитектуры, узкие места и риски.
- [drivers.md](drivers.md) — архитектурные драйверы, ограничения и приоритеты.
- [TO-BE.md](TO-BE.md) — целевое архитектурное видение на горизонт одного года.
- [diagrams/to-be-container.puml](diagrams/to-be-container.puml) — C4 Container
  диаграмма целевого решения.
- [adr/ADR-001-architecture-evolution.md](adr/ADR-001-architecture-evolution.md)
  — стратегия эволюции архитектуры.

## Ключевая идея

Для AdScale не подходит ни сохранение текущего монолита без изменений, ни
полная перепись за три месяца. Оптимальная стратегия — постепенная декомпозиция
по паттерну Strangler Fig: сначала вынести критичный путь ставок, затем отделить
событийную статистику, аналитику и финансовый контур.
